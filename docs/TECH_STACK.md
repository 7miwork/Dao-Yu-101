# Technology Stack

This document outlines the technologies used in the Dao-Yu-101 project.

## Current Technologies

### Core Framework
- **Astro** - Modern static site generator with island architecture
  - File-based routing
  - Component-based architecture
  - Server-side rendering
  - Static site generation

### Build & Deployment
- **GitHub Pages** - Static site hosting
- **Node.js** - Runtime environment
- **npm** - Package management

### Styling
- **Vanilla CSS** - Pure CSS for styling
  - CSS Grid and Flexbox
  - CSS animations
  - Responsive design principles

### Development Tools
- **TypeScript** - Type checking and development experience
- **ESLint** - Code linting
- **Prettier** - Code formatting

## Future Planned Technologies

### Internationalization
- **i18n support** - Multi-language capabilities
  - Language detection
  - Translation management
  - Locale-specific content

### Game UI Components
- **Canvas/WebGL** - For interactive game elements
- **Three.js** - 3D graphics rendering
- **Game physics libraries** - For interactive elements

### Voxel Themed Design
- **CSS 3D transforms** - For voxel-style visuals
- **SVG animations** - For smooth transitions
- **Particle effects** - For atmospheric elements

## Architecture Decisions

### Why Astro?
- Excellent performance with static generation
- Island architecture for optimal loading
- Great developer experience
- Strong TypeScript support
- Flexible component system

### Why Vanilla CSS?
- No framework overhead
- Full control over styling
- Better performance for static sites
- Easier to implement custom animations

### Future Considerations
- **Component libraries** - For consistent UI patterns
- **State management** - For interactive features
- **API integration** - For dynamic content
- **Progressive Web App features** - For enhanced user experience
